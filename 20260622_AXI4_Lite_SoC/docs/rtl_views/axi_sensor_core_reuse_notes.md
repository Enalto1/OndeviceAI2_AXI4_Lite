## `sr04.v`

Top module:

```verilog
module sr04 (
    input clk,
    input rst,
    input ultra_btn,
    input echo,
    output [8:0] distance,
    output trig
);
```

Important behavior:

- `ultra_btn` is a start request.
- `trig` is generated by `sr04_controller`.
- `distance` is latched when echo returns low after the response phase.
- `tick_gen_us` creates a 1 us tick with parameter `F_COUNT = 100_000_000 / 1_000_000`.

First wrapper mapping:

- `CONTROL[0]` enables SR04 command writes.
- `COMMAND[0]` generates one-clock `sr04_start_pulse`.
- `SR04_VALUE[8:0]` reads `distance`.
- `STATUS[0]` reads live `trig`.

## `dht11.v`

Top module:

```verilog
module dht11 (
    input clk,
    input rst,
    input dht_btn,
    output led,
    output [7:0] hm,
    output [7:0] tm,
    inout dht11
);
```

Important behavior:

- `dht_btn` is a start request.
- `led` is assigned from the controller `valid` signal.
- `hm` and `tm` latch humidity and temperature when `valid` is true.
- `dht11` is bidirectional and driven low by the controller during the start sequence.
- `tick_gen_us_dht11` creates a 1 us tick with parameter `F_COUNT = 100_000_000 / 1_000_000`.

First wrapper mapping:

- `CONTROL[8]` enables DHT command writes.
- `COMMAND[8]` generates one-clock `dht_start_pulse`.
- `DHT_VALUE[7:0]` reads `hm`.
- `DHT_VALUE[15:8]` reads `tm`.
- `STATUS[8]` reads live `led` as `dht_valid_live`.

## Reuse Decision

Both source files can be directly instantiated by `axi_sensor_core`. No adapted Sensor copy is required for the first wrapper. Full DHT11 protocol verification may be deferred to a richer future testbench, but the interface is sufficiently clear for RTL implementation.

## Prompt 10 Boundary

Reference sensor RTL remains unchanged. No UVM, MicroBlaze software, or Vivado block design is created in Prompt 10.

## Phase B Implementation Decision

Phase B implemented the AXI4-Lite Sensor wrapper at:

```text
rtl_work/axi_peripherals/axi_sensor_core/hdl/axi_sensor_core.v
```

Implementation facts:

- The wrapper instantiates `sr04` as `u_sr04`.
- The wrapper instantiates `dht11` as `u_dht11`.
- `CONTROL[0]` enables `COMMAND[0]` to create one-clock `sr04_start_pulse`.
- `CONTROL[8]` enables `COMMAND[8]` to create one-clock `dht_start_pulse`.
- `SR04_VALUE` exposes `sr04.distance`.
- `DHT_VALUE` exposes `dht11.hm` and `dht11.tm`.
- `STATUS[0]` exposes live `sr04.trig`.
- `STATUS[8]` exposes live `dht11.led` as `dht_valid_live`.
- No adapted Sensor copy was created.
- Reference Sensor RTL remains unchanged.
- FND, GPIO, Timer, SPI, I2C, UVM, MicroBlaze software, and block-design files are not instantiated or created by this wrapper.
